UUID types according to RFC 4122

Version 1 - time based

1ed54304 - 2e98 - 11ed - a261 - 0242ac120002
tltltltltltltltl - tmtmtmtm - vththth - xchchclcl - nnnnnnnnnnnn

tl - time_low - Least significant bits (highest change frequency)

tm - time mid - Middle bits

th - time_hi - Most significant bits of timestamp

v - version

v and th are labeled together as time_hi_and_version in the RFC

tl tm th - 60 bits in total, count of 100-nanosecond intervals since 00:00:00.00, 15 October 1582

x - variant

ch - clock_seq_hi - Clock sequence can be incremented to generate UUIDs faster than the timestamp ticks

cl - clock_seq_low

n - node - IEEE 802 MAC address

Version 2 - DCE security version

00000001 - 2e98 - 21ed - a201 - 0242ac120002
idididididididid - tmtmtmtm - vththth - xchchdmdm - nnnnnnnnnnnn

dm - Local domain

id - Local identifier

See specification at https://pubs.opengroup.org/onlinepubs/9696989899/chap5.htm#tagcjh_08_02_01_01

Version 3 - Name-based, MD5

8925cceb - 1898 - 3997 - 9aac - 2574f4cf32f4
h5h5h5h5h5h5h5h5 - h5h5h5h5 - vh5h5h5 - xh5h5h5h5 - h5h5h5h5h5h5h5h5h5h5h5h5

h5 - MD5 digest of namespace (UUID) and name

Only 121 or 122 bits are used from the MD5 digest - the version and variant bits are replaced
